# 요구사항 정의서

> **프로젝트**: GCMediAI 회사소개 홈페이지 리뉴얼
> **비전**: Beyond EMR, Toward Medical OS
> **작성일**: 2026-03-16

---

## 프로젝트 개요

GC메디아이는 국내 최대 의료 IT 네트워크(25,000처+)를 보유한 기업으로, AI 중심의 의료 플랫폼 기업으로 도약하는 비전을 대외에 명확히 전달하는 공식 홈페이지를 구축한다.

**목표**
- GC메디아이의 브랜드 정체성(Medical OS) 전달
- 주요 사업영역(의사랑, 유팜, UBIST, 똑닥, 미소몰) 소개
- 투자자(IR) 및 미디어(NEWS) 대상 정보 제공
- 파트너/고객의 문의 창구 마련

---

## 페이지별 요구사항

### 메인 (/)

#### Must Have
- 히어로 섹션: "Beyond EMR, Toward Medical OS" 메시지 및 브랜드 비주얼
- AI Capability 수치 표시: 의료기관 25,000처+, 누적 데이터 2억 건+, H100 기반 Medical LLM
- 사업분야 요약 카드 (4개 영역 링크)
- 최신 뉴스 미리보기 (최근 3~5건)
- IR 주요 지표 미리보기 (정적 데이터)
- CTA 버튼 (회사소개, 사업분야, 문의하기 이동)

#### Should Have
- 수치 카운트업 애니메이션
- 스크롤 트리거 페이드인 애니메이션
- 사업분야 hover 인터랙션

#### Nice to Have
- 배경 영상 또는 파티클 애니메이션
- 실시간 주가 위젯

---

### 회사소개 (/about)

#### Must Have
- 기업 소개 3단 내러티브 (현재 → 전환 → 미래)
- 연혁 타임라인
- 임직원/조직 현황 (기본 정보)

#### Should Have
- 가족사/관계사 소개 (로고 + 간략 설명)
- 오시는 길 (지도 임베드, 주소/교통 안내)

#### Nice to Have
- 인터랙티브 타임라인 (연도 클릭 필터)
- 경영진 프로필 상세 페이지

---

### 사업분야 (/business)

#### Must Have
- 4개 영역 탭/섹션 구성
  - 메디컬 인프라: 의사랑(EMR), 유팜(약국 솔루션)
  - 데이터·마케팅: UBIST (의료 데이터 분석)
  - 플랫폼: 똑닥 (병원 예약/접수)
  - 유통·의료기기: 미소몰 (의료 소모품/기기)
- 각 영역별 주요 기능 및 고객 가치 설명
- 각 서비스 공식 사이트 외부 링크

#### Should Have
- 서비스별 주요 수치/실적 표시
- 사업 영역 간 연계성 시각화 (인포그래픽)

#### Nice to Have
- 서비스별 데모/소개 영상 임베드
- 고객사 로고 슬라이더

---

### IR (/ir)

IR 페이지는 재무정보, 공시자료, 주주공고, NEWS 4개 섹션으로 구성된다. 주가정보는 Should Have로 분류되며 향후 추가한다.

> **데이터 소스**: 재무정보·공시·뉴스 데이터는 `src/lib/data/` 폴더의 정적 JSON 파일로 관리한다.

#### Must Have
- 정적 재무정보 표시 (최근 연도 요약)
- 공시자료 목록 (링크 제공)
- 주주공고 섹션
- NEWS 섹션: 보도자료 목록 (제목, 날짜, 미리보기)
- NEWS 섹션: 기사 상세 또는 외부 링크 연결

#### Should Have
- 주가정보 섹션 (정적 또는 API 연동) — MVP 이후 추가
- 재무정보 연도별 비교 표/차트
- 공시자료 PDF 다운로드 링크
- NEWS 카테고리/태그 필터링
- NEWS 페이지네이션 또는 무한 스크롤
- NEWS 날짜순 정렬

#### Nice to Have
- 실시간 주가 API 연동 (예: 한국거래소 또는 증권사 API)
- 재무 데이터 인터랙티브 차트 (Chart.js / Recharts)
- 뉴스 CMS 연동 (예: Notion, Sanity, Contentful)
- OG 이미지 자동 생성
- RSS 피드 제공

---

### 문의하기 (/contact)

#### Must Have
- 문의 폼 (이름, 이메일, 문의 유형 선택, 제목, 내용)
- 문의 유형 선택: 사업제휴 / 채용 / 언론 / 기타
- 필수 항목 유효성 검사 (클라이언트 사이드)
- 제출 성공/실패 피드백 메시지
- MVP 제출 처리: Server Action으로 제출 데이터를 서버 로그에 기록. 이메일 발송은 MVP 이후 구현 (아래 Should Have 참고)

#### Should Have
- 이메일 발송 기능 (서버 액션 또는 API Route)
- 스팸 방지 (reCAPTCHA 또는 honeypot)

#### Nice to Have
- 자동 응답 이메일 발송
- 문의 내역 관리 어드민 페이지

## 공통 요구사항

### 레이아웃

#### Must Have
- 공통 헤더 (로고, GNB: 회사소개 / 사업분야 / IR / 문의하기)
- 공통 푸터 (회사명, 주소, 대표번호, 사업자등록번호, 저작권, 개인정보처리방침 링크, 패밀리사이트 링크)
- 스크롤 시 헤더 고정(sticky) 처리
- 404 에러 페이지: 브랜드 일관된 디자인 + 메인으로 이동 링크
- 500 서버 에러 페이지: 브랜드 일관된 디자인 + 메인으로 이동 링크
- 개인정보처리방침 페이지 (`/privacy`): 법적 의무 충족을 위한 별도 페이지

#### Should Have
- 모바일 햄버거 메뉴
- 현재 페이지 GNB 활성화 표시
- 페이지 내 앵커 이동 스무스 스크롤

#### Nice to Have
- 상단으로 이동 버튼 (Scroll To Top)
- 사이드 퀵 메뉴

---

### 반응형

#### Must Have
- 3개 브레이크포인트 지원
  - 모바일: ~767px
  - 태블릿: 768px~1199px
  - 데스크톱: 1200px+
- 모든 페이지 모바일 레이아웃 최적화
- 터치 인터랙션 지원

#### Should Have
- 이미지 반응형 처리 (Next.js Image 컴포넌트 활용)
- 폰트 크기 반응형 조정

---

### SEO

#### Must Have
- 페이지별 `<title>`, `<meta description>` 설정
- Open Graph (OG) 태그 (title, description, image)
- `robots.txt`, `sitemap.xml` 생성
- 시맨틱 HTML (h1~h6 계층 구조, landmark 요소)

#### Should Have
- 구조화 데이터 (JSON-LD: Organization, WebSite)
- Canonical URL 설정

#### Nice to Have
- 다국어 hreflang 태그
- 동적 OG 이미지 생성 (Next.js `opengraph-image`)

---

### 성능

#### Must Have
- Lighthouse 성능 점수 80점 이상 목표 (Core Web Vitals 포함)
  - LCP (최대 콘텐츠 페인트) < 2.5s
  - CLS (누적 레이아웃 이동) < 0.1
  - FID/INP < 200ms
- Next.js Image 컴포넌트로 이미지 최적화
- 코드 스플리팅 (App Router 기본 제공)
- 폰트 최적화 (`next/font` 활용)

#### Should Have
- 불필요한 서드파티 스크립트 지연 로딩

#### Nice to Have
- Edge Runtime 활용
- 이미지 CDN 연동

---

### 접근성

#### Must Have
- 키보드 내비게이션 지원 (모든 인터랙티브 요소 Tab 접근 가능)
- 이미지 alt 텍스트 작성 (모든 `<img>`, `next/image` 포함)
- 폼 레이블과 오류 메시지의 스크린 리더 지원 (`aria-describedby`, `role="alert"`)
- 색상 대비 기준 충족 (4.5:1 이상)

#### Should Have
- WCAG 2.1 AA 수준 전면 준수 (GC메디아이가 공공기관 대상 서비스를 운영하는 경우 Must Have로 상향 검토)

## MVP 범위

### 포함 범위 (Must Have 전체)

- **메인 (/)**: 히어로, AI 수치, 사업분야 요약 카드, 뉴스 미리보기, IR 정적 미리보기, CTA
- **회사소개 (/about)**: 기업 소개 내러티브, 연혁
- **사업분야 (/business)**: 4개 영역(메디컬 인프라: 의사랑·유팜 / 데이터·마케팅: UBIST / 플랫폼: 똑닥 / 유통·의료기기: 미소몰) 소개
- **IR (/ir)**: 정적 재무정보, 공시자료 목록, 주주공고, NEWS(보도자료 목록 + 상세)
- **문의하기 (/contact)**: 문의 폼 (문의 유형 선택 포함, 클라이언트 유효성 검사, Server Action 서버 로그 기록)
- **공통**: 헤더(로고+GNB), 푸터(사업자등록번호·개인정보처리방침 포함), 반응형(모바일/태블릿/데스크톱), 기본 SEO(meta, OG, sitemap, robots), 개인정보처리방침 페이지(/privacy), 에러 페이지(404/500)

### 제외 범위 (MVP 이후)

| 항목 | 이유 |
|------|------|
| IR 실시간 주가 API 연동 | 외부 API 계약 및 설정 필요 |
| 뉴스 CMS 연동 | CMS 도입 의사결정 필요 |
| 문의 이메일 발송 서버 | 메일 서버/서비스 설정 필요 |
| 오시는 길 지도 | 지도 API 키 발급 필요 |
| 스크롤 애니메이션 | 핵심 기능 아님, 폴리싱 단계 |
| Google Analytics / Vercel Analytics | 도메인 확정 후 적용. Vercel Analytics(무료 내장)를 우선 적용 후 GA 추가 검토 |
| 다국어 지원 (영문) | 번역 리소스 미확보 |
| 다크모드 | 디자인 가이드 확장 필요 |
| IR NEWS CMS 연동 | 콘텐츠 수급 방식 미확정 |
