모든 페이지(에러 페이지 포함)에 아래 두 컴포넌트가 래핑된다.

| 컴포넌트 | 경로                                      | 역할                                                                                  |
| -------- | ----------------------------------------- | ------------------------------------------------------------------------------------- |
| `Header` | `src/components/layout/Header/Header.tsx` | 로고, GNB(회사소개·사업분야·IR·문의하기), 모바일 햄버거 메뉴, sticky 처리             |

```typescript
// src/app/ir/news/[slug]/page.tsx
export async function generateMetadata({ params }: Props) {
  const news = getNewsBySlug(params.slug);
  return {
    title: `${news.title} | GC메디아이 IR`,
    openGraph: { title: news.title, description: news.preview },
  };
}
```

### 필요한 컴포넌트

| 컴포넌트 | 설명                                                             |
| -------- | ---------------------------------------------------------------- |
| `Button` | "메인으로 이동" CTA (`primary` variant) — `next/link`로 `/` 이동 |

### 비고

- 브랜드 컬러(`$color-primary`, `$color-background`) 유지 — 디자인 일관성
- `src/app/not-found.tsx`는 공통 레이아웃(`layout.tsx`)을 자동 상속

## 10. 500 에러 페이지

| 항목       | 내용                                                               |
| ---------- | ------------------------------------------------------------------ |
| **URL**    | 서버 에러 발생 시                                                  |
| **파일**   | `src/app/error.tsx`                                                |
| **렌더링** | 클라이언트 컴포넌트 (`'use client'` 필수 — Next.js error.tsx 규칙) |
| **인증**   | 불필요                                                             |

### 레이아웃 구성

```
Header (공통)
│
└── 에러 콘텐츠 섹션
    ├── 에러 안내 메시지 ("일시적인 오류가 발생했습니다")
    ├── 재시도 Button (reset 함수 호출)
    └── 메인으로 이동 Button
│
Footer (공통)
```

### 필요한 컴포넌트

| 컴포넌트 | 설명                                                                                           |
| -------- | ---------------------------------------------------------------------------------------------- |
| `Button` | 재시도 버튼 (`secondary` variant, Next.js `reset()` 호출) + 메인 이동 버튼 (`primary` variant) |

### 비고

- Next.js App Router의 `error.tsx`는 반드시 `'use client'`
- `reset` prop(함수)을 받아 컴포넌트 트리 재시도 처리
