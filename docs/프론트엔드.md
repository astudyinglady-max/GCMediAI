# 프론트엔드 개발 가이드

## UI 컴포넌트 우선 사용 원칙

> **RULE**: 새 페이지·섹션 개발 시 아래 등록된 UI 컴포넌트를 **반드시 먼저** 사용한다.
> 네이티브 HTML 요소(`<button>`, `<select>`, `<input>`, `<input type="checkbox">` 등)를 직접 쓰는 것은 금지.
> 컴포넌트에 없는 기능이 필요하면 컴포넌트를 확장하거나, 새 컴포넌트를 `src/components/ui/` 하위에 추가한다.

---

## 등록된 UI 컴포넌트

### `Button` — `src/components/ui/Button/Button.tsx`

범용 버튼. **submit, 링크 제외 모든 버튼**에 사용.

```tsx
import Button from '@/components/ui/Button/Button';

// variant: 'solid' | 'outline' | 'ghost'  (기본: 'solid')
// size: 'sm' | 'md' | 'lg'               (기본: 'md')

<Button variant="solid" size="md" onClick={handler}>확인</Button>
<Button variant="outline" size="sm" disabled>비활성</Button>
<Button type="submit" variant="solid" size="md">문의 보내기</Button>
```

| variant | 용도 |
|---------|------|
| `solid` | 주요 CTA (네이비 배경, 흰 텍스트) |
| `outline` | 보조 액션 |
| `ghost` | 미묘한 액션, 테이블/카드 내부 |

---

### `Dropdown` — `src/components/ui/Dropdown/Dropdown.tsx`

커스텀 셀렉트 드롭다운. **`<select>` 대신 항상 사용**.
controlled 컴포넌트이므로 서버 액션에 값을 넘길 때는 **hidden input 패턴** 사용.

```tsx
import Dropdown from '@/components/ui/Dropdown/Dropdown';

// options: { value: string; label: string }[]
// value: 현재 선택값 (state)
// onChange: (value: string) => void
// placeholder?: string   (기본: '선택하세요')
// disabled?: boolean

const [selected, setSelected] = useState('');

<Dropdown
  options={[
    { value: 'a', label: '옵션 A' },
    { value: 'b', label: '옵션 B' },
  ]}
  value={selected}
  placeholder="선택해주세요"
  onChange={setSelected}
/>

{/* 서버 액션(form)에 전달 필요 시 */}
<input type="hidden" name="fieldName" value={selected} />
```

### `Checkbox` — `src/components/ui/Checkbox/Checkbox.tsx`

커스텀 체크박스. **`<input type="checkbox">` 대신 항상 사용**.
네이티브 input을 래핑하므로 `name`, `required`, `onChange` 등 표준 props 모두 지원.

```tsx
import Checkbox from '@/components/ui/Checkbox/Checkbox';

// label?: string  — 체크박스 우측 텍스트
// 나머지 props: InputHTMLAttributes<HTMLInputElement> 전부 지원

<Checkbox
  name="agree"
  required
  label="이용약관에 동의합니다."
/>

<Checkbox
  checked={isChecked}
  onChange={(e) => setIsChecked(e.target.checked)}
  label="전체 선택"
/>
```

### `Input` — `src/components/ui/Input/Input.tsx`

텍스트 입력 필드. **`<input type="text|email|tel|…">` 대신 항상 사용**.
value가 있을 때 오른쪽에 X 버튼이 나타나 한 번에 초기화할 수 있다.
controlled / uncontrolled 모두 지원하며 표준 `InputHTMLAttributes` props 전부 사용 가능.

```tsx
import Input from '@/components/ui/Input/Input';

// type?: 'text' | 'email' | 'tel' | 'password' | 'search' | 'url' | 'number'  (기본: 'text')
// onClear?: () => void  — X 버튼 클릭 시 추가 콜백 (controlled 모드에서 state 초기화 등)
// 나머지 props: InputHTMLAttributes<HTMLInputElement> 전부 지원

// ── controlled ──
const [name, setName] = useState('');

<Input
  value={name}
  onChange={(e) => setName(e.target.value)}
  onClear={() => setName('')}
  placeholder="홍길동"
/>

// ── uncontrolled (form name 기반) ──
<Input
  name="phone"
  type="tel"
  required
  placeholder="010-0000-0000"
/>
```

| 동작 | 설명 |
|------|------|
| 값 없음 | 일반 input, X 버튼 미노출 |
| 값 있음 | 오른쪽에 X(초기화) 버튼 노출 |
| disabled | X 버튼 미노출, 필드 비활성 스타일 |
| onClear 미전달 | 내부적으로 onChange 호출로 값 초기화 |

## 레이아웃 공통 컴포넌트

> **RULE**: 서브 페이지(about / business / ir / contact 등) 상단 히어로와 탭 내비게이션은
> 반드시 아래 두 컴포넌트를 사용한다. 각 페이지마다 별도로 구현하는 것을 금지한다.

### `PageHero` — `src/components/layout/PageHero/PageHero.tsx`

서브 페이지 상단 히어로(서브 헤더) 영역. **새 페이지 추가 시 반드시 사용**.

```tsx
import PageHero from '@/components/layout/PageHero/PageHero';

// eyebrow: 영문 소제목 (예: "About GC MediAI")
// title: 한국어 메인 제목
// description?: 설명 문구 (string 또는 ReactNode)
// backgroundImage?: 배경 이미지 URL (About 페이지처럼 선택적 사용)

<PageHero
  eyebrow="About GC MediAI"
  title="회사소개"
  description="27년간 대한민국 의료 현장을 함께해온 의료 IT 전문 기업"
/>

{/* 배경 이미지가 필요한 경우 */}
<PageHero
  eyebrow="About GC MediAI"
  title={COMPANY_INFO.vision}
  description="..."
  backgroundImage="https://example.com/image.jpg"
/>

{/* description에 JSX가 필요한 경우 */}
<PageHero
  eyebrow="Contact"
  title="문의하기"
  description={
    <>
      첫 번째 줄<br />두 번째 줄
    </>
  }
/>
```

### `SubTabNav` — `src/components/layout/SubTabNav/SubTabNav.tsx`

탭 내비게이션. `'use client'` 컴포넌트. **탭이 있는 페이지에서 반드시 사용**.
`top: 72px`에 sticky 고정, 하단 인디케이터 포함.

```tsx
import SubTabNav from '@/components/layout/SubTabNav/SubTabNav';
import type { TabItem } from '@/components/layout/SubTabNav/SubTabNav';

// tabs: { id: string; label: string }[] — 탭 목록
// activeTab: string — 현재 활성 탭 id
// onTabChange: (id: string) => void — 탭 변경 콜백

const TABS: TabItem[] = [
  { id: 'overview', label: '개요' },
  { id: 'history', label: '연혁' },
];
const [activeTab, setActiveTab] = useState('overview');

<SubTabNav
  tabs={TABS}
  activeTab={activeTab}
  onTabChange={setActiveTab}
/>
```

## 컴포넌트 선택 흐름

```
UI 요소 필요
    │
    ▼
등록된 컴포넌트로 구현 가능한가?
    │
   YES ──────────────────────────────────► 해당 컴포넌트 사용
    │
    NO
    │
    ▼
기존 컴포넌트 props 확장으로 해결 가능한가?
    │
   YES ──────────────────────────────────► 컴포넌트 수정 후 사용
    │
    NO
    │
    ▼
src/components/ui/ 하위에 새 컴포넌트 생성
(네이밍: PascalCase 폴더 + 동일명 .tsx / .module.scss)
```

## 폼(Form) 구현 시 체크리스트

- [ ] 텍스트 입력 → `Input` (clear 버튼 내장)
- [ ] 드롭다운 → `Dropdown` + hidden input (서버 액션 연동 시)
- [ ] 체크박스 → `Checkbox`
- [ ] 라디오 → `Radio`
- [ ] 제출 버튼 → `Button type="submit"`
- [ ] 개인정보 동의 필요 시 → `Checkbox required` + 약관 텍스트 박스

---

## 새 컴포넌트 추가 규칙

- 위치: `src/components/ui/{ComponentName}/`
- 파일 구성: `ComponentName.tsx` + `ComponentName.module.scss`
- `'use client'` 필수 (인터랙션 컴포넌트)
- props 타입은 명시적으로 선언, `any` 금지
- 기존 컴포넌트 스타일 변수(`src/styles/variables.scss`) 재사용
