## 개요

이 문서는 개발팀이 `src/styles/_variables.scss`, `_mixins.scss`, `_typography.scss`를 작성할 때 참조하는 디자인 토큰 및 컴포넌트 규칙의 단일 진실 공급원(Single Source of Truth)이다. 디자이너-개발자 간 해석 차이를 최소화하고 WCAG 2.1 AA 접근성 기준 준수를 보장한다.

## 1. 컬러 시스템

### 1-1. 브랜드 팔레트

| SCSS 변수           | Hex       | 이름            | 주요 용도                                |
| ------------------- | --------- | --------------- | ---------------------------------------- |
| `$color-primary`    | `#F13C05` | Brand Orange    | 로고, CTA 버튼, 메인 슬로건, 핵심 셰이프 |
| `$color-secondary`  | `#111111` | Typography Navy | 본문 텍스트, 내비게이션, 경계선          |
| `$color-accent`     | `#1E6BFF` | Electric Blue   | 메인 일러스트레이션, 액센트 요소         |
| `$color-background` | `#F3F3F3` | Neutral BG      | 전체 페이지 배경                         |
| `$color-surface`    | `#FFFFFF` | White           | 카드 배경, 모달, 입력 필드 배경          |
| `$color-highlight`  | `#3F3AA8` | Deep Violet     | 그래픽 버블, 시각적 깊이, 강조 키워드    |

### 1-2. 시맨틱 컬러

| SCSS 변수        | Hex       | 용도                        |
| ---------------- | --------- | --------------------------- |
| `$color-success` | `#2E7D32` | 성공 메시지, 완료 상태      |
| `$color-warning` | `#E65100` | 경고 메시지, 주의 표시      |
| `$color-error`   | `#C62828` | 오류 메시지, 폼 유효성 실패 |
| `$color-info`    | `#01579B` | 안내 메시지, 정보 강조      |

> 시맨틱 컬러는 `$color-surface(#FFFFFF)` 위에서 사용하며, 모두 WCAG 2.1 AA 기준(4.5:1)을 충족한다.

### 1-3. 텍스트 컬러 단계

| SCSS 변수         | Hex       | 대비율 (on `$color-background`) | 용도                                                 |
| ----------------- | --------- | ------------------------------- | ---------------------------------------------------- |
| `$text-primary`   | `#111111` | ~10.8:1 ✅ AAA                  | 제목, 본문 주요 텍스트                               |
| `$text-secondary` | `#4A5C7A` | ~5.5:1 ✅ AA                    | 본문 보조 텍스트, 레이블                             |
| `$text-disabled`  | `#9E9E9E` | ~2.8:1 ⚠️                       | 비활성 텍스트 (인터랙티브 요소 아닌 장식용에만 사용) |
| `$text-inverse`   | `#FFFFFF` | —                               | 어두운 배경 위 텍스트                                |

### 1-4. WCAG 2.1 AA 대비율 검증

> 요구사항: 색상 대비 4.5:1 이상 (일반 텍스트), 3:1 이상 (대형 텍스트 18px bold+ 또는 24px+ 및 UI 컴포넌트)

| 전경색                     | 배경색                      | 대비율     | AA 일반 | AA 대형/UI | 비고                                                                    |
| -------------------------- | --------------------------- | ---------- | ------- | ---------- | ----------------------------------------------------------------------- |
| `$color-secondary #111111` | `$color-background #F3F3F3` | **10.8:1** | ✅      | ✅         | 본문 텍스트로 권장                                                      |
| `$color-secondary #111111` | `$color-surface #FFFFFF`    | **11.9:1** | ✅      | ✅         | 카드 내부 텍스트 사용 가능                                              |
| `$color-primary #F13C05`   | `$color-background #F3F3F3` | **3.40:1** | ❌      | ✅         | **일반 텍스트 사용 금지**, CTA 버튼·아이콘·대형 텍스트(18px bold+) 전용 |
| `$color-primary #F13C05`   | `$color-surface #FFFFFF`    | **3.75:1** | ❌      | ✅         | **일반 텍스트 사용 금지**, CTA 버튼·아이콘·대형 텍스트(18px bold+) 전용 |
| `$text-inverse #FFFFFF`    | `$color-primary #F13C05`    | **3.75:1** | ❌      | ✅         | CTA 버튼 텍스트 (18px bold 이상 권장)                                   |
| `$color-accent #1E6BFF`    | `$color-background #F3F3F3` | **3.87:1** | ❌      | ✅         | **버튼 레이블·아이콘·액센트 요소**에만 허용                             |
| `$color-accent #1E6BFF`    | `$color-surface #FFFFFF`    | **4.27:1** | ❌      | ✅         | **버튼 레이블·아이콘·액센트 요소**에만 허용                             |
| `$color-highlight #3F3AA8` | `$color-background #F3F3F3` | **7.22:1** | ✅      | ✅         | 강조 키워드·버블에 사용 가능                                            |
| `$color-highlight #3F3AA8` | `$color-surface #FFFFFF`    | **7.95:1** | ✅      | ✅         | 강조 키워드·버블에 사용 가능                                            |
| `$text-inverse #FFFFFF`    | `$color-highlight #3F3AA8`  | **7.95:1** | ✅      | ✅         | 헤더 텍스트 사용 가능                                                   |
| `$text-secondary #4A5C7A`  | `$color-background #F3F3F3` | **5.5:1**  | ✅      | ✅         | 보조 텍스트·레이블 사용 가능                                            |

**⚠️ 핵심 제약사항**: `$color-primary(#F13C05)`와 `$color-accent(#1E6BFF)`는 일반 본문 텍스트(16px 이하 regular)에 사용 금지. CTA 버튼 텍스트는 반드시 18px bold 이상 또는 `$text-inverse(#FFFFFF)`를 사용한다.

### 1-5. 60-30-10 배색 규칙

| 비중             | 컬러                                                   | 적용 예시                          |
| ---------------- | ------------------------------------------------------ | ---------------------------------- |
| **60%** — 지배색 | `$color-background (#F3F3F3)`                          | 전체 페이지 배경, 여백             |
| **30%** — 보조색 | `$color-secondary (#111111)`                           | 헤더, 텍스트, 경계선, 내비게이션   |
| **10%** — 강조색 | `$color-primary (#F13C05)` + `$color-accent (#1E6BFF)` | CTA 버튼, 액센트 요소, 핵심 아이콘 |

## 3. 간격 시스템

### 3-1. 기본 단위

**4px 기본 단위**를 사용한다. 모든 간격은 4의 배수여야 한다.

```scss
// src/styles/_variables.scss
$spacing-base: 4px;

$spacing-1: 4px; //  4px — 아이콘 내부 패딩, 인라인 갭
$spacing-2: 8px; //  8px — 텍스트와 아이콘 간격, 태그 패딩
$spacing-3: 12px; // 12px — 버튼 내부 수직 패딩 (sm)
$spacing-4: 16px; // 16px — 버튼 내부 수직 패딩 (md), 카드 내부 패딩 기본
$spacing-5: 20px; // 20px — 입력 필드 패딩
$spacing-6: 24px; // 24px — 카드 내부 패딩 (표준), 섹션 내 요소 간격
$spacing-7: 32px; // 32px — 컴포넌트 간 간격
$spacing-8: 40px; // 40px — 섹션 내부 하위 블록 간격
$spacing-9: 48px; // 48px — 모바일 섹션 상하 패딩
$spacing-10: 64px; // 64px — 태블릿 섹션 상하 패딩
$spacing-11: 80px; // 80px — 데스크톱 섹션 상하 패딩
$spacing-12: 96px; // 96px — 대형 섹션 분리
$spacing-13: 120px; // 120px — 히어로 섹션 상하 패딩
```

### 3-2. 용도별 가이드

| 상황                      | 권장 변수                   | 값      |
| ------------------------- | --------------------------- | ------- |
| 아이콘 ↔ 텍스트 간격      | `$spacing-2`                | 8px     |
| 버튼 내부 수평 패딩 (md)  | `$spacing-6`                | 24px    |
| 버튼 내부 수직 패딩 (md)  | `$spacing-4`                | 16px    |
| 카드 내부 패딩            | `$spacing-6`                | 24px    |
| 카드 그리드 갭            | `$spacing-6` ~ `$spacing-7` | 24~32px |
| 폼 필드 간 수직 간격      | `$spacing-6`                | 24px    |
| 섹션 내 하위 블록 간격    | `$spacing-7` ~ `$spacing-8` | 32~40px |
| 섹션 상하 패딩 (모바일)   | `$spacing-9`                | 48px    |
| 섹션 상하 패딩 (태블릿)   | `$spacing-10`               | 64px    |
| 섹션 상하 패딩 (데스크톱) | `$spacing-11`               | 80px    |
| 히어로 섹션 상하 패딩     | `$spacing-13`               | 120px   |

## 4. 브레이크포인트 & 반응형

### 4-1. 브레이크포인트 정의

```scss
// src/styles/_variables.scss
$breakpoint-mobile: 767px; // ~767px: 모바일
$breakpoint-tablet: 768px; // 768px~1199px: 태블릿
$breakpoint-desktop: 1200px; // 1200px+: 데스크톱

$container-max-width: 1200px;
$container-padding-mobile: $spacing-4; // 16px
$container-padding-tablet: $spacing-8; // 40px
$container-padding-desktop: $spacing-11; // 80px (양쪽 각각)
```

### 4-2. 반응형 믹스인

```scss
// src/styles/_mixins.scss

// 모바일 이하 (~767px)
@mixin mobile {
  @media (max-width: #{$breakpoint-mobile}) {
    @content;
  }
}

// 태블릿 이상 (768px+)
@mixin tablet {
  @media (min-width: #{$breakpoint-tablet}) {
    @content;
  }
}

// 태블릿만 (768px~1199px)
@mixin tablet-only {
  @media (min-width: #{$breakpoint-tablet}) and (max-width: #{$breakpoint-desktop - 1px}) {
    @content;
  }
}

// 데스크톱 이상 (1200px+)
@mixin desktop {
  @media (min-width: #{$breakpoint-desktop}) {
    @content;
  }
}
```

**사용 예시**:

```scss
// HeroSection.scss
.heroSection {
  padding: $spacing-9 $container-padding-mobile; // 모바일 기준

  @include tablet {
    padding: $spacing-10 $container-padding-tablet;
  }

  @include desktop {
    padding: $spacing-13 $container-padding-desktop;
  }
}
```

### 4-3. 컨테이너 믹스인

```scss
// src/styles/_mixins.scss
@mixin container {
  width: 100%;
  max-width: $container-max-width;
  margin-inline: auto;
  padding-inline: $container-padding-mobile;

  @include tablet {
    padding-inline: $container-padding-tablet;
  }

  @include desktop {
    padding-inline: $container-padding-desktop;
  }
}
```

## 7. 컴포넌트 디자인 규칙

### 7-1. 버튼 (Button)

**파일**: `src/components/ui/Button/Button.scss`

#### 변형 (Variant)

| Variant     | 배경                       | 텍스트                       | 테두리           | 용도                                 |
| ----------- | -------------------------- | ---------------------------- | ---------------- | ------------------------------------ |
| `primary`   | `$color-primary` (#F13C05) | `$text-inverse` (#FFFFFF)    | 없음             | CTA, 핵심 액션 (18px bold 이상 권장) |
| `secondary` | `transparent`              | `$color-secondary` (#111111) | `$border-strong` | 보조 액션                            |
| `ghost`     | `transparent`              | `$color-secondary` (#111111) | 없음             | 텍스트성 버튼, 내비게이션            |

#### 크기 (Size)

| Size | 높이   | 수평 패딩           | 폰트 크기 | 반경                |
| ---- | ------ | ------------------- | --------- | ------------------- |
| `sm` | `36px` | `$spacing-4 (16px)` | `14px`    | `$radius-md (8px)`  |
| `md` | `44px` | `$spacing-6 (24px)` | `16px`    | `$radius-lg (12px)` |
| `lg` | `52px` | `$spacing-8 (40px)` | `18px`    | `$radius-lg (12px)` |

> **접근성**: 최소 터치 영역 44×44px 확보 (WCAG 2.1 AA 2.5.5). `sm` 버튼은 아이콘 전용이거나 터치 영역을 padding으로 확장한다.

#### 상태별 스타일

```scss
// primary 버튼 예시
.button {
  &.primary {
    background: $color-primary;
    color: $text-inverse;
    transition:
      background $transition-base,
      transform $transition-fast;

    &:hover {
      background: darken($color-primary, 8%); // #C43004
    }

    &:active {
      transform: scale(0.98);
    }

    &:focus-visible {
      outline: 2px solid $border-focus;
      outline-offset: 2px;
    }

    &:disabled,
    &.loading {
      background: rgba($color-primary, 0.4);
      cursor: not-allowed;
      transform: none;
    }
  }
}
```

#### 로딩 상태

- `loading` prop 전달 시 텍스트 숨김, 스피너 아이콘 표시
- 스피너: `border`와 `border-top`을 활용한 CSS 회전 애니메이션 (`@keyframes spin`)
- 버튼 너비는 로딩 중에도 변하지 않도록 최소 너비 고정

### 7-4. 모달 (Modal)

모달은 현재 폴더구조에 명시적으로 없으나, 사업분야 상세나 공시 자료 미리보기에 필요할 수 있다. 도입 시 다음 규칙을 따른다.

#### 오버레이

- 배경: `rgba(0, 0, 0, 0.5)`
- 진입 애니메이션: `opacity 0 → 1`, `200ms ease`
- 클릭 시 모달 닫힘

#### 모달 패널

- 최대 너비: `640px` (중형), `480px` (소형)
- 반경: `$radius-xl (16px)`
- 내부 패딩: `$spacing-8 (40px)` (데스크톱), `$spacing-6 (24px)` (모바일)
- 진입 애니메이션: `translateY(20px) → translateY(0)`, `300ms cubic-bezier(0.34, 1.56, 0.64, 1)`
- 모바일에서는 바텀 시트(bottom sheet) 패턴 검토

#### 접근성 규칙

- `role="dialog"`, `aria-modal="true"`, `aria-labelledby` 필수
- 열릴 때 포커스를 모달 내 첫 인터랙티브 요소로 이동
- **포커스 트랩**: Tab 키가 모달 내 요소만 순환
- **ESC 키**: 모달 닫힘
- 닫힐 때 포커스를 모달을 연 트리거 요소로 복귀

## 10. 아이콘 시스템

### 10-1. 라이브러리

| 항목                 | 내용                      |
| -------------------- | ------------------------- |
| **아이콘 셋**        | coolicons                 |
| **Iconify 프리픽스** | `ci`                      |
| **총 아이콘 수**     | 442개                     |
| **패키지**           | `@iconify/react`          |
| **설치**             | `pnpm add @iconify/react` |

coolicons는 선 굵기가 일정한 outline 스타일 아이콘 셋으로, DREAM BUBBLE 팔레트의 Brand Orange / Typography Navy / Electric Blue 기반의 활기차고 현대적인 분위기와 잘 어울린다.

### 10-2. 사용법

```tsx
// 기본 사용 (크기는 SCSS 클래스로 제어)
import { Icon } from '@iconify/react';

<Icon icon="ci:arrow-right-md" className={styles.iconMd} />;
```

#### SCSS 아이콘 크기 클래스

`Icon` 컴포넌트는 기본적으로 `1em` 크기로 렌더링된다. 크기는 아래 전용 클래스로만 지정한다. **`width`/`height` 인라인 스타일 금지.**

```scss
// src/styles/_mixins.scss 또는 globals.scss
.iconSm {
  width: 16px;
  height: 16px;
  flex-shrink: 0;
}
.iconMd {
  width: 20px;
  height: 20px;
  flex-shrink: 0;
}
.iconLg {
  width: 24px;
  height: 24px;
  flex-shrink: 0;
}
.iconXl {
  width: 32px;
  height: 32px;
  flex-shrink: 0;
}
```

| 클래스   | 크기 | 주요 용도                                        |
| -------- | ---- | ------------------------------------------------ |
| `iconSm` | 16px | Badge 내 아이콘, 에러 메시지 접두 아이콘         |
| `iconMd` | 20px | 버튼 내 아이콘, 입력 필드 내 아이콘              |
| `iconLg` | 24px | 섹션 소제목 옆 아이콘, 카드 헤더 아이콘          |
| `iconXl` | 32px | 사업분야 카드 대표 아이콘, 빈 상태 일러스트 대체 |

#### 컬러

`Icon` 컴포넌트는 SVG를 `color: currentColor`로 렌더링한다. **별도 color prop이나 인라인 스타일 사용 금지.** 부모 요소의 SCSS 텍스트 컬러가 자동 상속된다.

```scss
// ✅ 올바른 방법 — 부모 컬러 상속
.iconWrapper {
  color: $color-secondary; // Icon이 #111111 색으로 렌더링됨
}

// ✅ 버튼 내 아이콘 — 버튼 텍스트 컬러 자동 상속
.button.primary {
  color: $text-inverse; // 버튼 텍스트와 아이콘 모두 흰색
}

// ❌ 금지 — 인라인 스타일
<Icon icon="ci:arrow-right-md" style={{ color: '#111111' }} />
```

#### 접근성

```tsx
// 장식용 아이콘 (텍스트 옆에 위치, 의미 없음) → aria-hidden
<Icon icon="ci:arrow-right-md" className={styles.iconMd} aria-hidden="true" />

// 단독 아이콘 버튼 (텍스트 없음) → aria-label 필수
<button aria-label="메뉴 닫기">
  <Icon icon="ci:close-md" className={styles.iconLg} aria-hidden="true" />
</button>

// 상태 아이콘 (성공/에러 등) → role="img" + aria-label
<Icon icon="ci:circle-check" className={styles.iconMd} role="img" aria-label="성공" />
```

## 11. 금지 사항

### CLAUDE.md 상속 금지 사항

- 인라인 스타일(`style=""`) 사용 금지 — 모든 스타일은 SCSS로 작성
- CSS 클래스명에 kebab-case 금지 — camelCase 사용 (`heroSection`, `cardWrapper`)
- `!important` 사용 금지 — 단, `prefers-reduced-motion` 오버라이드 한정 허용

### 아이콘 관련 금지 사항

- **coolicons(ci) 외 다른 아이콘 셋 혼용 금지** — 시각 일관성 유지
- **`<Icon>` 컴포넌트에 `style` prop 직접 전달 금지** — 크기/컬러 모두 SCSS 클래스로 제어
- **`width`/`height` 어트리뷰트 인라인 지정 금지** — `iconSm`/`iconMd`/`iconLg`/`iconXl` 클래스 사용
- **장식용 아이콘에 `aria-label` 단독 부여 금지** — 텍스트 옆 아이콘은 `aria-hidden="true"` 사용
- **아이콘만 있는 버튼에 `aria-label` 누락 금지** — 스크린 리더 접근성 필수
- **프로젝트 아이콘 할당표(10-3)에 없는 아이콘 임의 추가 금지** — 추가 필요 시 할당표 먼저 업데이트

### 디자인 관련 추가 금지 사항

- **`$color-primary(#F13C05)` / `$color-accent(#1E6BFF)`를 일반 본문 텍스트(16px 이하 regular)에 사용 금지** — 대비율 미충족(3.4~3.9:1)
- **`$text-disabled` 컬러를 인터랙티브 요소(버튼, 링크)에 사용 금지** — WCAG 1.4.3 위반
- **그림자 없이 배경색만으로 카드 구분 금지** — 저시력 사용자 접근성 저하
- **색상만으로 상태 구분 금지** — 아이콘, 패턴, 텍스트를 병행 사용 (에러 상태: 빨간색 + 에러 아이콘 + 텍스트)
- **4px 배수가 아닌 임의 간격 값 하드코딩 금지** — 반드시 `$spacing-*` 변수 사용
- **Pretendard 외 웹폰트 추가 사용 금지** — 성능 요구사항(LCP < 2.5s) 충족을 위해
- **`z-index` 임의 숫자 사용 금지** — 아래 레이어 체계 준수

### z-index 레이어 체계

```scss
// src/styles/_variables.scss
$z-base: 0;
$z-above: 1;
$z-dropdown: 100;
$z-sticky: 200; // 헤더 sticky
$z-overlay: 300; // 모달 오버레이
$z-modal: 400; // 모달 패널
$z-toast: 500; // 토스트 알림
```
