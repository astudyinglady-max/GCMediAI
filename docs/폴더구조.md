## 핵심 폴더 역할 요약

| 폴더 | 역할 |
|------|------|
| `src/app/` | App Router 페이지 및 API Route Handler — URL 구조와 1:1 대응 |
| `src/components/ui/` | Button, Card 등 여러 페이지에서 재사용되는 기본 UI 컴포넌트 |
| `src/components/layout/` | 모든 페이지에 공통으로 들어가는 헤더, 푸터, 네비게이션 |
| `src/components/sections/` | 각 페이지를 구성하는 섹션 단위 컴포넌트 (페이지당 1~5개) |
| `src/lib/data/` | MVP 단계에서 DB 대신 사용하는 정적 콘텐츠 (TS 상수 또는 JSON) |
| `src/lib/actions/` | Next.js Server Actions — 문의 폼 제출 등 서버 사이드 로직 |
| `src/lib/utils/` | 날짜·숫자 포맷 등 UI에 종속되지 않는 순수 함수 |
| `src/lib/constants/` | 라우트 경로, 사이트명 등 앱 전역에서 사용하는 상수 |
| `src/styles/` | 글로벌 CSS 리셋, 디자인 토큰 변수, 반응형 믹스인 |
| `src/types/` | 도메인별 TypeScript 타입 정의 (컴포넌트와 분리) |
| `public/` | 이미지, 폰트 등 빌드 없이 그대로 서빙되는 정적 파일 |
| `docs/` | 기획 문서 모음 — 개발 코드와 무관, 의사결정 맥락 보존용 |
| `.github/workflows/` | PR 자동 품질 검사 (ESLint, 타입 체크, 빌드) |

## 컴포넌트 네이밍 규칙

```
컴포넌트명/
├── 컴포넌트명.tsx     # default export 컴포넌트
└── 컴포넌트명.scss    # 해당 컴포넌트 전용 스타일
```

- 폴더명과 파일명을 동일하게 유지하여 import 경로가 명확하다.
- 하나의 폴더에 하나의 컴포넌트만 배치한다 (단일 책임).
- SCSS 클래스명은 camelCase를 사용한다 (`heroSection`, `cardWrapper`).

## 데이터 흐름

```
정적 콘텐츠 (MVP)
  src/lib/data/*.ts  →  Server Component  →  브라우저

동적 콘텐츠 (Should Have)
  외부 API (DART 등)  →  src/app/api/*/route.ts  →  Client Component (TanStack Query)  →  브라우저

폼 제출
  브라우저 Form  →  src/lib/actions/contact.ts (Server Action)  →  Resend API
```
